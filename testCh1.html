第一章 問題：雲概念、架構與設計（含答案與說明）

1. Matthew 正在審查他的組織計劃採用的一項新的雲服務。在此服務中，雲提供商將在多租戶模型下創建虛擬伺服器實例。每個伺服器實例將僅對 Matthew 的公司可訪問。正在使用哪種雲部署模型？

  1. 混合雲
  2. 公有雲
  3. 私有雲
  4. 社群雲

答案： B

說明： 回答這個問題的關鍵是認識到多租戶模型涉及許多不同的客戶訪問託管在共享硬體上的雲資源。這使得它成為一個公有雲部署，無論對特定伺服器實例的訪問是否僅限於 Matthew 的公司。在私有雲部署中，只有 Matthew 的公司可以訪問託管在同一物理硬體上的任何資源。這不是多租戶。沒有跡象表明 Matthew 的組織正在結合公有雲和私有雲計算的資源，這將是混合雲，或者資源使用僅限於特定群組的成員，這將是社群雲。

2. Zeke 負責對從他組織數據中心的伺服器中移除的一組固態硬碟 (SSD) 進行清理。這些硬碟將在不同的專案中重複使用。以下哪種清理技術最有效？

  1. 加密擦除
  2. 物理銷毀
  3. 消磁
  4. 覆寫

答案： A

說明： 加密擦除是一種強大的清理技術，涉及使用強加密引擎加密數據，然後獲取該過程中生成的金鑰，使用不同的加密引擎對其進行加密，並銷毀第二輪加密產生的金鑰。這種技術對磁性硬碟和固態硬碟都有效。消磁和覆寫對 SSD 無效。物理銷毀會有效地清理媒體，但會阻止 Zeke 重複使用硬碟。

3. Tina 希望使用一種技術，讓她能夠捆綁工作負載並輕鬆地在不同的操作系統之間移動它們。哪種技術最能滿足這個需求？

  1. 虛擬機器
  2. 無伺服器計算
  3. Hypervisors
  4. 容器

答案： A

說明： 容器不提供簡單的可移植性，因為它們依賴於主機操作系統。Hypervisors 用於在設備上託管虛擬機器，所以這是另一個錯誤的答案。無伺服器計算是一種平台即服務模型，允許雲客戶在提供商的平台上運行自己的代碼而無需配置伺服器，所以這也是不正確的。虛擬機器是自包含的，並擁有自己的內部操作系統，因此可以在不同的主機操作系統之間移動它們。

4. 在雲參考架構下，以下哪項活動通常不屬於客戶的責任？

  1. 監控服務
  2. 準備系統
  3. 執行業務管理
  4. 處理問題報告

答案： B

說明： 在雲參考架構下，客戶的活動包括使用雲服務、執行服務試用、監控服務、管理服務安全、提供計費和使用報告、處理問題報告、管理租戶、執行業務管理、選擇和購買服務以及請求審計報告。準備系統是雲服務提供商的責任之一。

5. Seth 正在幫助他的組織將其 Web 伺服器集群遷移到雲提供商。此舉的目標是讓集群能夠根據不斷變化的需求進行擴展和縮減。Seth 希望實現雲計算的哪個特性？

  1. 可擴展性
  2. 按需自助服務
  3. 彈性
  4. 廣泛的網路訪問

答案： C

說明： 實際上 Seth 很可能希望實現所有這些目標，但最相關的是彈性。彈性是指系統根據當前需求水平動態增長和縮減的能力。可擴展性是指系統隨著需求增加而增長的能力，但不需要縮減的能力。

6. Sherry 正在為她的組織部署零信任網路架構。在這種方法中，以下哪個特性在驗證登錄嘗試時最不重要？

  1. 用戶身份
  2. IP 地址
  3. 地理位置
  4. 請求訪問的性質

答案： B

說明： 零信任網路架構的定義特徵是信任決策不基於網路位置，例如 IP 地址。使用其他特性是適當的，例如用戶的身份、請求訪問的性質以及用戶的地理（而非網路！）位置。

7. 以下哪種 hypervisor 模型最能抵抗攻擊？

  1. Type 1
  2. Type 2
  3. Type 3
  4. Type 4

答案： A

說明： 如果雲提供商能夠在 hypervisor 類型之間進行選擇，則裸機 (Type 1) hypervisor 優於運行在 OS 之上 (Type 2) 的 hypervisor，因為它提供的攻擊面較小。Type 3 和 4 hypervisor 不存在。

8. Joe 正在使用一個運行在公有雲提供商上的虛擬伺服器實例，並希望限制從網際網路訪問該伺服器的端口。哪種安全控制最能幫助他滿足這個需求？

  1. 地理圍欄
  2. 流量檢查
  3. 網路防火牆
  4. 網路安全組

答案： D

說明： 網路安全組為雲託管的伺服器實例提供相當於網路防火牆的功能。它們允許限制可能到達伺服器實例的流量。Joe 將無法修改網路防火牆規則，因為這些規則僅對雲提供商可用。地理圍欄會限制用戶可以訪問伺服器的地理位置，這不是 Joe 的要求。流量檢查可用於檢查到達實例的流量，但通常不用於創建基於端口的限制。

9. 以下哪種網路安全威脅最不可能直接影響對象儲存服務？

  1. 磁碟故障
  2. 用戶錯誤
  3. 勒索軟體
  4. 病毒

答案： D

說明： 對象儲存服務容易受到可能無意中銷毀或修改數據的磁碟故障和用戶錯誤的影響。它們也容易受到勒索軟體攻擊，這些攻擊會感染有權訪問對象儲存的系統，然後加密儲存在服務上的數據。它們不太可能受到傳統病毒的影響，因為它們沒有運行時環境。

10. Vince 希望在具有訪問敏感雲服務權限的用戶離開定義的物理區域時立即收到警報。他應該實施哪種類型的安全控制？

  1. 入侵防禦系統
  2. 地理圍欄
  3. 防火牆規則
  4. 地理標記

答案： B

說明： 地理圍欄可用於在用戶或設備離開定義的地理區域時觸發操作，例如警報。防火牆和入侵防禦系統可能會將地理資訊納入其決策過程，但不會提供 Vince 想要的即時通知。地理標記只是用執行操作的用戶的地理位置來註釋日誌記錄或其他數據，但不會直接根據地理位置提供警報。

11. 以下哪個特性不是雲計算標準定義的組成部分？

  1. 廣泛的網路訪問
  2. 快速配置
  3. 多租戶
  4. 按需自助服務

答案： C

說明： 雲計算是一個模型，用於實現對可配置計算資源（例如，網路、伺服器、儲存、應用程式和服務）共享池的普遍、方便、按需的網路訪問，這些資源可以通過最少的管理工作或服務提供商互動來快速配置和釋放。此定義不包括多租戶，這是公有雲計算的一個特性，但不是所有雲計算模型的特性。

12. 以下哪個來源提供了一套用於雲安全的廠商中立設計模式？

  1. 雲安全聯盟 (Cloud Security Alliance)
  2. 亞馬遜網路服務 (Amazon Web Services)
  3. 微軟 (Microsoft)
  4. (ISC)²

答案： A

說明： 雲安全聯盟 (CSA) 提供了一個企業架構參考指南，為雲安全提供廠商中立的設計模式。亞馬遜網路服務 (AWS) 和微軟確實提供雲設計模式，但它們是特定於這些供應商的服務產品。(ISC)² 不提供雲設計模式。

13. Lori 正在使用 API 訪問儲存在雲服務中的敏感資訊。Lori 正在進行哪個雲安全數據生命週期活動？

  1. 儲存 (Store)
  2. 使用 (Use)
  3. 銷毀 (Destroy)
  4. 創建 (Create)

答案： B

說明： 使用 API 是在生命週期的「使用」階段以編程方式訪問數據的一個例子。如果 Lori 只是將數據放入雲服務或在那裡維護數據，那將是「儲存」階段的一個例子。Lori 沒有創建或銷毀數據；她只是在使用已儲存在雲服務中的數據。

14. Helen 希望在雲中配置一個可以從伺服器掛載的磁碟卷。她想要什麼雲能力？

  1. 虛擬化伺服器
  2. 對象儲存
  3. 網路容量
  4. 塊儲存

答案： D

說明： 這是一個塊儲存的例子，即作為磁碟卷可用的儲存。對象儲存將文件保存在儲存桶中。虛擬化伺服器是計算能力，而不是儲存能力。網路容量用於將伺服器相互連接以及連接到網際網路，不用於儲存數據。

15. Ben 正在使用 sudo 命令在 Linux 伺服器上執行操作。他正在使用哪種類型的訪問？

  1. 服務訪問
  2. 未經授權的訪問
  3. 用戶訪問
  4. 特權訪問

答案： D

說明： sudo 命令允許普通用戶帳戶執行管理命令，是特權訪問的一個例子，而不是標準用戶訪問。在場景中沒有跡象表明 Ben 缺乏對此訪問的適當授權。服務訪問是指系統服務對資源的訪問，而不是個人。

16. 當設備丟失或被盜時，以下哪個密碼學目標可以防範由此帶來的風險？

  1. 不可否認性
  2. 身份驗證
  3. 完整性
  4. 機密性

答案： D

說明： 當設備丟失或被盜時，最大的風險是設備上包含的敏感數據落入壞人之手。機密性可以防範這種風險。不可否認性是指消息的接收者可以向第三方證明發送者的身份。身份驗證是證明身份的一種手段。完整性表明資訊自傳輸以來未被修改。

17. 在嘗試評估故障對客戶信心的影響時，哪種類型的業務影響分析工具最合適？

  1. 定量分析
  2. 定性分析
  3. 年度損失預期
  4. 單次損失預期

答案： B

說明： 定性工具通常用於業務影響評估中，以捕捉對無形因素（如客戶信心、員工士氣和聲譽）的影響。定量工具（例如計算年度損失預期和單次損失預期）僅適用於易於量化的風險。

18. Robert 正在審查一個已根據通用準則 (Common Criteria) 分配了 EAL2 評估保證等級的系統。他對該系統可能擁有的最高保證等級是什麼？

  1. 它已經過功能測試。
  2. 它已經過結構化測試。
  3. 它已經過正式驗證、設計和測試。
  4. 它已經過半形式化設計和測試。

答案： B

說明： EAL2 保證適用於系統經過結構化測試的情況。它是通用準則下的第二低保證等級。

19. Jake 希望使用第三方平台自動在雲服務提供商之間移動工作負載。哪種類型的工具最能滿足這個需求？

  1. 雲訪問安全代理 (Cloud access service broker)
  2. 資料庫
  3. 虛擬化
  4. 編排 (Orchestration)

答案： D

說明： 編排工具旨在管理工作負載並在雲服務提供商之間無縫地移動它們。虛擬化平台允許雲提供商託管虛擬伺服器實例，但它們不提供在不同提供商之間遷移工作負載的能力。資料庫是一種雲服務產品，允許有組織地儲存關係數據。雲訪問安全代理 (CASB) 允許在雲提供商之間一致地執行安全策略。

20. Robert 負責保護用於處理信用卡資訊的系統。哪個安全控制框架應該指導他的行動？

  1. HIPAA
  2. PCI DSS
  3. SOX
  4. GLBA

答案： B

說明： 支付卡行業數據安全標準 (PCI DSS) 管轄信用卡資訊的儲存、處理和傳輸。健康保險可攜性與責任法案 (HIPAA) 管轄受保護的健康資訊。薩班斯-奧克斯利法案 (SOX Act) 規範上市公司財務報告。格雷姆-里奇-比利勒法案 (GLBA) 保護個人財務資訊。

21. 哪種類型的努力試圖將組織的所有雲活動置於更集中的控制之下？

  1. 雲訪問安全代理
  2. 雲編排
  3. 雲治理
  4. 雲遷移

答案： C

說明： 雲治理計劃試圖將組織的所有雲活動置於更集中的控制之下。它們作為一個篩選機構，幫助確保組織使用的雲服務滿足技術、功能和安全要求。它們還提供了一個集中的監控點，用於監控重複服務，防止不同的業務部門在可以通過整合降低成本和操作環境複雜性的情況下花錢購買類似的服務。雲編排工具旨在管理工作負載並在雲服務提供商之間無縫地移動它們。雲訪問安全代理 (CASB) 允許在雲提供商之間一致地執行安全策略。雲遷移是從本地環境到雲環境或在兩個雲環境之間的過渡。

22. Chris 正在為他的公司設計一個內部使用的密碼系統。該公司有 1000 名員工，他們計劃使用非對稱加密系統。他們希望設置該系統，以便任何一對任意用戶都可以私密通信。他們總共需要多少個金鑰？

  1. 500
  2. 1,000
  3. 2,000
  4. 4,950

答案： C

說明： 非對稱密碼系統為每個用戶使用一對金鑰。在這種情況下，有 1000 個用戶，系統將需要 2000 個金鑰。

23. Erin 擔心她的組織使用的雲提供商會失敗，因此她正在制定一個策略，該策略將結合來自多個公有雲提供商的資源。哪個術語最能描述這個策略？

  1. 社群雲
  2. 多雲
  3. 私有雲
  4. 混合雲

答案： B

說明： 使用多個公有雲提供商以實現多樣性被稱為多雲策略。這就是 Erin 正在創建的場景。社群雲是共享的雲資源，對親和群組的成員開放。私有雲資源僅限單個組織使用。混合雲策略結合了公有雲和私有雲資源，而不是來自多個公有雲提供商的資源。

24. 以下哪項通常被認為是雲服務提供商的應用能力？

  1. 網路容量
  2. 託管電子郵件
  3. 塊儲存
  4. 無伺服器計算

答案： B

說明： 電子郵件是一種應用程式級服務，由雲提供商作為軟體即服務 (SaaS) 能力提供。塊儲存和網路容量是基礎設施即服務 (IaaS) 產品，是基礎設施能力。無伺服器計算是平台即服務 (PaaS) 產品，是平台能力。

25. 由於並非所有用戶都會同時訪問其服務產品的全部容量，雲提供商能夠進行什麼活動？

  1. 超額訂閱 (Oversubscription)
  2. 過度配置 (Overprovisioning)
  3. 配置不足 (Underprovisioning)
  4. 訂閱不足 (Undersubscription)

答案： A

說明： 超額訂閱意味著雲提供商可以向客戶出售總容量超過其基礎設施實際物理容量的容量，因為從宏觀角度來看，客戶永遠不會同時使用所有容量。訂閱不足是指雲提供商不出售其所有可用容量，這不需要用戶不同時訪問服務。過度配置是指客戶（而非服務提供商）購買了超出其需求的容量。類似地，配置不足是指客戶未購買足夠的容量來滿足其需求。

26. Brian 最近加入了一個組織，該組織的大部分服務運行在其自有數據中心的虛擬化平台上，但也利用 IaaS 提供商託管其 Web 服務和 SaaS 電子郵件系統。哪個術語最能描述該組織使用的雲環境類型？

  1. 公有雲
  2. 專用雲
  3. 私有雲
  4. 混合雲

答案： D

說明： 這個場景描述了公有雲和私有雲服務的混合。這是一個混合雲環境的例子。

27. 在基礎設施即服務 (IaaS) 環境中，供應商向客戶提供儲存服務的訪問權限，通常由誰負責從停用的硬碟中移除敏感數據？

  1. 客戶的安全團隊
  2. 客戶的儲存團隊
  3. 客戶的供應商管理團隊
  4. 供應商

答案： D

說明： 在基礎設施即服務環境中，安全職責遵循共享責任模型。由於供應商負責管理儲存硬體，供應商將保留銷毀或擦除停用硬碟的責任。然而，客戶仍有責任在利用供應商的儲存服務之前驗證供應商的清理程序是否滿足其要求。

28. Lucca 正在審查他組織的災難恢復流程數據，並注意到該企業主要網站的最大容忍停機時間 (MTD) 為兩小時。當他進行測試和驗證時，他對該網站的恢復時間目標 (RTO) 有何了解？

  1. 它需要少於兩小時。
  2. 它需要至少兩小時。
  3. MTD 太短，需要更長。
  4. RTO 太短，需要更長。

答案： A

說明： 當 Lucca 審查恢復時間目標 (RTO) 數據時，他需要確保組織能夠在基於兩小時最大容忍停機時間 (MTD) 的中斷後不到兩小時內恢復。

29. Alice 和 Bob 希望使用非對稱密碼系統相互通信。他們位於國家的不同地區，但已通過使用由相互信任的證書頒發機構簽名的數字證書交換了加密金鑰。

39. 您正在審查一份服務級別協議 (SLA)，發現其中一條規定保證您計劃使用的服務具有 99.99% 的正常運行時間。哪個術語最能描述這種類型的規定？

  3. 隱私性
  4. 彈性 (Resiliency)

81. 加密是保障雲端操作安全的重要工具。雖然可以加密雲端中的每個系統、數據片段和事務，但為什麼這對組織來說可能不是最佳選擇？

  1. 金鑰長度差異不提供任何實際的額外安全性。
  2. 它會導致額外的處理開銷和時間延遲。
  3. 它可能導致廠商鎖定。
  4. 數據主體可能會對此感到不滿。

答案： B

說明： 加密會消耗處理能力和時間；與所有安全控制一樣，額外的安全性意味著可衡量的運營能力降低——安全性和生產力之間總是有權衡。選項 A 是胡言亂語，只是一個干擾項。選項 C 是不正確的，因為廠商鎖定不是由加密引起的；如果雲提供商在持有您的數據時倒閉，可能會發生這種情況。數據主體是組織持有其個人身份資訊 (PII) 的個人；通常，他們不會知道或關心某些內容是否已加密（除非該 PII 發生洩露，然後調查人員會想確定該 PII 如何受到保護），並且可能歡迎完全加密，即使這可能意味著運營能力下降。

82. __________ 是盡職調查 (due care) 的範例，而 ___________ 是審慎查核 (due diligence) 的範例。

  1. 隱私數據安全策略；審計隱私數據安全策略規定的控制措施
  2. 歐盟通用數據保護條例 (GDPR)；格雷姆-里奇-比利勒法案 (GLBA)
  3. 門鎖；旋轉柵門
  4. 周界防禦；內部防禦

答案： A

說明： 盡職調查是履行職責所需的最低努力水平；在雲安全中，這通常是雲客戶為了保護其擁有的數據而必須展示的謹慎程度。審慎查核是在支持或促進盡職調查方面採取的任何活動。這個答案因此是最佳的：盡職調查由策略規定，而支持該策略的活動（此處為審計策略要求的控制措施）是審慎查核的證明。通用數據保護條例 (GDPR) 和 GLBA 都是立法授權；這些可能規定了盡職調查的標準，但它們本身不是盡職調查或審慎查核。門鎖和旋轉柵門是物理安全控制；它們都可能是盡職調查努力的例子，但都不能證明審慎查核。盡職調查和審慎查核可以通過內部或外部控制/流程來證明；在控制位於何處方面沒有區別。

83. 以下哪項是機密計算環境的關鍵組件？

  1. TEE
  2. TPM
  3. HSM
  4. PKI

答案： A

說明： 機密計算通過使用可信執行環境 (TEE) 來保護使用中的數據。機密計算環境也可能利用其他安全技術，包括可信平台模塊 (TPM)、硬體安全模塊 (HSM) 和公鑰基礎設施 (PKI)，但這些技術不保護使用中的數據，因此不是機密計算環境所必需的。

84. 以下哪個計劃為可能在政府環境中使用的計算硬體提供通用認證過程？

  1. FedRAMP
  2. NIST 800-53
  3. 通用準則 (Common Criteria)
  4. FIPS 140-2

答案： C

88. 您的組織已遷移到平台即服務 (PaaS) 配置。雲提供商內部的網路管理員訪問了您的數據並將您的用戶列表出售給了競爭對手。根據所有適用法律，誰有義務進行數據洩露通知？

  1. 負責的網路管理員
  2. 雲提供商
  3. 監督您部署的監管機構
  4. 您的組織

答案： D

說明： 雲客戶最終對涉及數據安全和隱私的所有法律後果負責；雲提供商可能對與這些責任相關的財務成本負責，但這些損害賠償只能在雲客戶發出通知很久之後才能追回。所有其他選項都不正確，因為它們沒有正確識別根據所有適用法律有義務發出數據洩露通知的人。該責任落在雲客戶身上。

89. 如果一個組織希望在雲端中最大程度地保留對其資產的控制權，他們應該選擇哪種服務和部署模型組合？

  2. 基礎設施即服務 (IaaS)，混合雲
  3. 軟體即服務 (SaaS)，公有雲
  4. 基礎設施即服務 (IaaS)，私有雲

90. Henry 的公司部署了一個廣泛的物聯網 (IoT) 基礎設施用於建築監控，其中包括環境控制、佔用傳感器以及各種其他有助於管理建築的傳感器和控制器。在對物聯網部署的分析中，Henry 應將以下哪個安全問題報告為最關鍵？

  1. IoT 設備缺乏本地儲存空間來存放安全日誌，這在 IoT 設備中很常見。
  2. IoT 設備可能沒有單獨的管理介面，允許同一網路上的任何人嘗試登錄它們，從而使暴力攻擊成為可能。
  3. IoT 設備可能不支持強加密通信，從而使日誌和傳感器數據在網路上被攔截。
  4. IoT 設備的長期支持和修補模型可能帶來安全和運營風險。

答案： D

說明： Henry 最關心的應該是 IoT 設備的長期安全性和可支持性。隨著這些設備越來越多地嵌入建築物和基礎設施中，了解支持模型和安全模型非常重要。缺乏單獨的管理訪問權限和缺乏強加密都可以通過將 IoT 設備放置在專用子網或網路上來解決，從而阻止其他用戶直接訪問這些設備。這將有助於在不產生過多費用或複雜性的情況下限制風險，並且是一種常見的做法。最後，缺乏儲存空間可能是一個問題，但在查看 IoT 設備可能產生的風險時，這不是最重要的。

92. 哪個雲計算組件最容易受到逃逸攻擊？

99. 如果完全開發，哪種計算技術有可能破壞現代加密算法的安全性？

100. 通常認為業務連續性 (BC) 工作與災難恢復 (DR) 工作之間有何區別？

  1. BC 涉及恢復時間目標 (RTO)，而 DR 涉及恢復點目標 (RPO)。
  2. BC 適用於人為事件（如縱火或盜竊），而 DR 適用於自然災害。
  3. BC 旨在在正常運營中斷期間維持關鍵功能，而 DR 旨在在中斷後恢復正常運營。
  4. BC 涉及保護人力資產（人員、員工、用戶），而 DR 涉及保護財產（資產、數據）。

答案： C

說明： 從技術上講，BC 工作旨在確保關鍵業務功能在破壞性事件期間能夠繼續運行，而 DR 工作則旨在支持在事件後恢復正常運營。然而，在實踐中，這些努力通常會重合，使用相同的計劃/人員，並有許多相同的程序。選項 A 不正確；BC 和 DR 都使用 RTO 和 RPO 作為衡量成功的指標。選項 B 不正確；BC 和 DR 工作不特定於破壞性事件的原因。選項 D 不正確；健康和人類安全在所有安全工作中都應至關重要，只有極少數例外。
