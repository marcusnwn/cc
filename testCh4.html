第四章 問題：雲應用程式安全（含答案與說明）

1. Mikayla 希望驗證她從 GitHub 下載的軟體組件。當該組件作為其整合過程的一部分下載並包含在她的環境中時，她如何驗證底層軟體沒有安全缺陷？

  1. 驗證文件的校驗和。
  2. 驗證文件的簽名。
  3. 驗證文件的哈希值。
  4. Mikayla 無法通過所述選項確保沒有安全缺陷。

答案： D

說明： Mikayla 無法僅通過驗證軟體是否已簽名、其哈希是否與官方發布的哈希匹配，或校驗和是否匹配來驗證第三方軟體沒有漏洞或其他安全問題。她需要自己測試該軟體，或確定一個她可以信賴的第三方進行適當的軟體測試。即便如此，該軟體仍可能包含缺陷。

2. Lin 希望允許她的用戶在訪問她的服務時使用由第三方身份提供商提供的現有憑證。她需要從以下列表中提供哪個元素？

  1. 用戶 ID
  2. 身份驗證
  3. 授權
  4. 身份證明

答案： C

說明： Lin 需要提供使用其組織服務的授權。第三方身份提供商將提供用戶 ID 並進行身份驗證，然後向 Lin 提供令牌或驗證，證明用戶是他們聲稱的人。身份證明不一定由任何一方在此場景中提供，如果驗證某人是他們聲稱的人比僅僅允許訪問服務更重要，這可能會使接受第三方憑證變得具有挑戰性。

3. Joanna 的軟體供應商不向其客戶提供源代碼。在以下列表中，測試供應商軟體包安全的最佳選擇是什麼？

  1. 對軟體執行靜態分析。
  2. 實施配對編程技術。
  3. 審查軟體中是否存在硬編碼的秘密。
  4. 執行動態測試。

答案： D

說明： Joanna 知道她無法訪問源代碼，這意味著靜態分析不是一個選項，識別秘密可能會很困難。

4. 哪種 SDLC 模型最常與雲開發流程相關聯？

  1. 敏捷 (Agile)
  2. RAD
  3. 螺旋 (Spiral)
  4. 瀑布 (Waterfall)

答案： A

說明： 敏捷開發模型最常與雲服務和開發相關聯。快速應用開發、螺旋和瀑布方法更常與傳統開發環境相關聯，但可用於雲服務和系統。

5. Susan 希望避免雲應用程式開發中的常見陷阱。以下哪個陷阱經常與雲環境相關聯？

  1. 在雲中構建的應用程式的可靠性
  2. 在雲中構建的應用程式的可擴展性
  3. 在雲中構建的應用程式的冗餘性
  4. 在雲中構建的應用程式的安全性

答案： D

說明： Susan 知道雲服務將允許她在可靠、可擴展的基礎設施上構建她的應用程式，該基礎設施可以利用多種方式的冗餘。她還知道，無論代碼構建或運行的環境如何，保護代碼仍然是一個問題。

6. Susan 希望避免數據整合問題。她知道 OWASP 雲 Top 10 包括服務和數據整合安全問題，並且正在部署一個基於 REST 的 API 供其客戶在訪問其服務時使用。她正在使用 API 金鑰，但擔心第三方攔截和訪問數據。她應在其實現中包含什麼來解決這個問題？

  1. 數據標記化
  2. 確保靜態加密
  3. 確保傳輸中加密
  4. 數據遮蔽

答案： C

說明： Susan 知道傳輸中加密（通常通過 TLS）是防止第三方攔截和訪問通過 API 調用發送給其客戶的數據的正確解決方案。標記化和遮蔽用於保護應用程式中的數據，但它們不滿足防止攔截的需求，並且數據在場景描述中是在傳輸中而不是靜態的。

7. Ben 希望為他的軟體開發工作收集業務需求，並且正在使用敏捷方法。以下哪項 *不是* 敏捷流程中收集用戶需求的常用方法？

  1. 頭腦風暴
  2. 文件審查
  3. 用戶觀察
  4. 調查

答案： B

說明： 頭腦風暴、調查、研討會、用戶觀察和許多其他技術通常用於收集業務需求。然而，文件審查不涉及用戶需求或反饋。

8. 靜態加密是雲環境 SDLC 中包含的保護性設計元素，這是由於哪個常見的雲設計主題？

  1. 快速彈性
  2. 多租戶
  3. 計量服務
  4. 可擴展性

答案： B

說明： 由於多個租戶共享相同的底層基礎設施，靜態加密可防止無意中的數據暴露或重用基礎設施上的殘留數據。彈性、計量服務和可擴展性不需要靜態加密，實際上由於加密開銷，可能會導致成本略高。

9. Nick 希望避免他的 CI/CD 管道中的常見陷阱。以下哪個是可能損害雲開發工作的常見 CI/CD 陷阱？

  1. 流程自動化
  2. 使用指標
  3. 使用多個部署路徑
  4. 依賴版本控制系統

答案： C

說明： 擁有多個部署路徑意味著可以避免 CI/CD 系統中的元素或流程。單一路徑以及備份它的應急流程有助於避免此問題。自動化、使用指標和使用版本控制都是 CI/CD 管道的推薦最佳實踐。

10. Maria 希望將她現有的身份提供商與她的雲提供商的服務整合。大多數雲身份提供商整合使用哪個通用標準？

  1. IDPL
  2. OpenLDAP
  3. SAML
  4. ConnectID

答案： C

說明： Maria 知道 IdP 通常與 OpenID Connect 或 SAML 整合。OpenLDAP 是 LDAP 的開源實現，但由於它是目錄服務，因此不用於此類整合。ConnectID 和 IDPL 是為此問題編造的。

11. Jack 希望使他的團隊能夠開發雲原生應用程式。以下哪項 *不是* 雲原生應用程式設計的常見元素？

  1. 優化的彙編代碼
  2. 自動化發布管道
  3. 容器
  4. 微服務

答案： A

說明： Jack 知道最適合雲應用程式的語言才有意義，彙編語言不太可能適合他的 CI/CD 環境。使用自動化發布管道、容器和微服務都是雲應用程式開發和設計的常見元素。

12. 在軟體開發生命週期 (SDLC) 的哪個階段，用戶參與 *最* 關鍵？

  1. 定義
  2. 設計
  3. 開發
  4. 測試

答案： A

說明： 雖然某些開發模型允許用戶參與整個過程，但用戶輸入在定義階段最為必要，開發人員可以在此階段了解業務/用戶需求——系統/軟體實際應該在功能和性能方面產生什麼。在整個過程中進行參與是有幫助的，但如果沒有在定義階段的用戶輸入，整個過程將沒有正確的起點。

13. Brian 希望確保他在其開發過程中考慮到 OWASP Top 10 雲風險。他知道監管合規性在列表上，並希望將其包含在 SDLC 中。在 SDLC 的哪個階段考慮監管合規性最有意義？

  1. 分析和需求定義
  2. 設計
  3. 實施
  4. 測試

答案： A

說明： Brian 需要在 SDLC 的需求收集階段提出監管合規性問題。如果它不是一個要求，它就不會進入設計和實施階段。測試監管合規性可能具有挑戰性，因為並非所有合規性都可以通過技術手段進行測試。

14. Angie 組織的測試過程包括訪問設計規範、源代碼和正在運行的應用程式。她正在使用哪種類型的安全測試方法？

  1. 白盒
  2. 灰盒
  3. 紅盒
  4. 黑盒

答案： A

說明： Angie 正在使用白盒或全知測試方法。灰盒或部分知識測試提供對環境或系統的部分但非全部訪問權限。黑盒或零知識測試要求測試人員像沒有內部系統知識的攻擊者一樣行事。紅盒是為此問題編造的。

15. CWE/SANS Top 25 最危險軟體錯誤包括使用硬編碼憑證。可以使用哪個常見的雲服務組件來避免雲託管軟體和應用程式的這個問題？

  1. MFA 令牌
  2. TPM
  3. KMS
  4. API 金鑰

答案： C

說明： KMS 或金鑰管理服務用於安全地儲存和訪問秘密，允許在需要時使用它們而無需硬編碼。TPM 用於儲存硬體的秘密，MFA 令牌用於多因素身份驗證，API 金鑰用於安全地向 API 進行身份驗證和訪問 API，但它們都不提供避免雲應用程式中硬編碼憑證所需的功能。

16. Dana 的組織要求其部署的每個應用程式都有一個 SBOM。哪個 OWASP Top 10 項目有助於通過 SBOM 避免？

  1. 易受攻擊和過時的組件
  2. 損壞的訪問控制
  3. 注入
  4. 安全配置錯誤

答案： A

說明： 軟體物料清單 (SBOM) 用於識別軟體包中的每個組件。Dana 知道擁有 SBOM 將有助於她跟踪其軟體堆疊中每個組件的版本和可用更新。

17. Yun 工作的公司向客戶提供 API 訪問。Yun 希望對 API 訪問進行速率限制並收集計費資訊，同時使用中央授權和訪問管理系統。Yun 應部署哪種類型的工具來滿足這些要求？

  1. API 網關
  2. API 代理
  3. API 防火牆
  4. 下一代 API 管理器

答案： A

說明： API 網關旨在聚合 API 訪問、為 API 使用提供身份驗證、速率限制以及收集有關 API 使用情況的統計信息和數據。API 代理用於解耦軟體組件，API 防火牆是為保護特定 API 的 API 而構建的專用防火牆，下一代 API 管理器是為此問題編造的。

18. IAST 通常與 SDLC 的哪個階段相關聯？

  1. 設計
  2. 測試
  3. 實施
  4. 部署

答案： B

說明： 交互式應用程式安全測試 (IAST) 使用軟體工具監控應用程式運行情況並收集有關發生情況和軟體執行方式的信息。它通常在 SDLC 的 QA 或測試階段實施。

19. Gary 的雲服務向客戶提供對 API 的訪問。以下哪個是 API 中常見的安全缺陷？

  1. 使用非結構化數據
  2. 缺乏身份驗證
  3. 使用半結構化數據
  4. 缺乏加密

答案： B

說明： 未受保護的 API 是一個常見問題，缺乏身份驗證是一個主要問題。數據提供方式——結構化、非結構化或半結構化——不是安全問題。加密通常不由 API 提供，而是由 Web 伺服器或底層服務提供。

20. Dan 希望在其雲環境中加密靜態數據。在加密靜態數據時，他應尋找哪個加密標準？

  1. TLS
  2. AES-256
  3. SSL
  4. Blowfish

答案： B

說明： AES-256 是一個廣泛接受和使用的靜態數據加密標準。TLS 用於傳輸中加密。SSL 已過時，Blowfish 具有已知的漏洞，包括生日攻擊。

21. Mark 希望確保他的軟體供應商在其軟體驗證過程中使用了行業最佳實踐。他知道 NIST 為開發人員驗證代碼定義了許多推薦的最低標準。以下哪項 *不是* NIST 推薦的供應商或開發人員代碼驗證的最低標準？

  1. 使用自動化測試。
  2. 執行基於代碼的（靜態）分析。
  3. 僅檢查內部開發的軟體。
  4. 進行威脅建模。

答案： C

說明： 檢查包含的軟體是否是內部或外部開發的是最佳實踐，僅檢查您自己的組件通常會留下大量未經檢查的軟體包。您可以在 www.nist.gov/itl/executive-order-improving-nations-cybersecurity/recommended-minimum-standard-vendor-or-developer 找到完整列表。

22. Lori 希望確保其供應商提供的包含的軟體組件是安全的。她應使用哪種類型的過程來對這些軟體包進行評估？

  1. Web 應用程式漏洞掃描
  2. 軟體組成分析
  3. 漏洞掃描
  4. 版本號驗證過程

答案： B

說明： Lori 的最佳選擇是使用 SCA 工具執行軟體組成分析 (SCA) 過程，該工具可以識別開源庫和其他組件。漏洞掃描無法識別與軟體相關的所有組件，版本號驗證過程是為此問題編造的。

23. Christine 記錄了一個軟體測試用戶故事，內容如下：“作為攻擊者，我將上傳惡意軟體作為我的表單提交的一部分，這將利用解析用戶提交內容的解析軟體。” Christine 準備進行哪種類型的測試？

  1. 濫用案例測試
  2. 靜態測試
  3. QA 測試
  4. SCA 測試

答案： A

說明： Christine 正在準備進行濫用案例測試，該測試記錄了對軟體功能的濫用。靜態測試審查源代碼，QA 測試驗證軟體質量，SCA 是軟體組成分析，它著眼於軟體的底層組件。

24. OWASP 的應用程式安全驗證標準 (ASVS) 具有三個主要使用模型。以下哪個 *不是* 基於其設計目標的預期使用模型？

  1. 用作指標
  2. 用於審計
  3. 用作指導
  4. 在採購期間使用

答案： B

說明： ASVS 旨在用作評估 Web 應用程式中可信賴程度的指標，為開發人員提供如何內建安全控制的指導，以及在採購過程中指定應用程式安全驗證要求。它不是設計為審計標準。

25. Ian 希望使用特定於雲的應用程式問題列表。他應該選擇以下哪個選項？

  1. OWASP Top 10
  2. NIST Dirty Dozen
  3. SANS Top 25
  4. MITRE ATT&CK-RS

答案： A

說明： OWASP Top 10 是特定於雲的列表，而 SANS Top 25 則描述了一般的軟體錯誤。NIST Dirty Dozen 和 MITRE ATT&CK-RS 是為此問題編造的。

26. Nick 希望為其團隊的軟體版本控制使用通用格式。如果他想使用通用行業慣例，他應該使用哪種版本控制格式？

  1. Codename.version
  2. Major.build.minor.patch
  3. RFC number.version.patch
  4. Major.minor.patch

答案： D

說明： Major.minor.patch 是一種常見的版本控制格式。雖然沒有行業標準，但按比例順序排列版本號是一種常見做法。

27. 哪個術語用於描述產品所有軟體組件的列表？

  1. 組件索引
  2. SBOM
  3. 版本目錄
  4. SCCM

答案： B

說明： 軟體物料清單 (SBOM) 是軟體包或程序所有組件的列表，作為軟體安全程序的一部分，它被認為越來越重要。

28. Valerie 的公司最近遭受了針對他們使用的第三方應用程式的成功 SQL 注入攻擊。供應商尚未提供針對 SQL 注入漏洞的補丁，但 Valerie 由於業務需求需要保持該應用程式的生產狀態。Valerie 可以部署哪種類型的工具來防範對其 Web 應用程式的 SQL 注入攻擊？

  1. DAM
  2. WAF
  3. XML 防火牆
  4. API 網關

答案： B

說明： Web 應用程式防火牆 (WAF) 通常具有針對 SQL 注入等常見攻擊的內建保護。Valerie 應選擇一個 WAF 來防範她已知的 SQL 注入以及針對 Web 應用程式的其他常見攻擊。

29. Jackie 希望允許應用程式使用其所需的庫和其他依賴項運行，而無需為每個應用程式設置獨立的操作系統。她應該使用哪種技術來讓她輕鬆地在不同操作系統之間移動應用程式包？

  1. 包
  2. 容器
  3. 虛擬機器
  4. Hypervisors

答案： B

說明： 容器允許輕鬆移動應用程式，因為它們包含應用程式所需的依賴項和組件，而無需打包完整的操作系統。包是軟體組件，虛擬機器運行在 hypervisor 上。

30. Charles 使用他組織的憑證登錄，並且能夠在各種系統和應用程式中使用該登錄。Charles 使用的是哪種技術？

  1. SAML
  2. SSO
  3. OpenID Connect
  4. OTP

答案： B

說明： Charles 正在使用單一登錄 (SSO) 技術，允許他一次登錄即可使用許多服務和系統。SAML 和 OpenID Connect 用於聯合，而 OTP 是一次性密碼。

31. Henry 希望確保只有授權客戶才能使用其組織面向公眾的 API。此目的常用的安全技術是什麼？

  1. API 金鑰
  2. 單一登錄
  3. API 聯合
  4. 複雜的 API 密碼

答案： A

說明： Henry 的組織應使用 API 金鑰來確保只有授權用戶才能使用他們向客戶公開的 API。

32. Nancy 希望確保她的組織不會因其軟體的許可問題而遇到麻煩，並且她知道供應商使用許可證伺服器控制訪問，每個安裝都會與之聯繫。為了確保她將來不會在某個時候遇到服務中斷，她應特別注意以下哪個方面？

  1. 許可證期限
  2. 許可證條款是否可以披露
  3. 許可證成本
  4. 合同中包含的第三方子許可證

答案： A

97. Malika 希望確保人為錯誤不會影響其組織秘密的安全性。以下哪個做法將最有效地防止與人相關的問題影響其秘密安全？

98. Frank 知道他的組織打算將聯合身份作為其雲服務環境的一部分。他應確保其現有的本地身份管理系統支持哪個標準來提供幫助？

99. James 在其開發過程的核心使用 CI/CD 管道。他應使用哪個設計模式來確保其 QA 過程不影響生產？

  1. 將正在進行 QA 的軟體添加到其生產環境以允許實時測試。
  2. 為 QA 測試創建新環境，然後在測試後提升到生產環境。
  3. 為 QA 測試複製生產環境，然後在測試後提升到生產環境。
  4. 將軟體添加到 QA 環境進行測試，然後允許生產用戶訪問帶有工具的 QA 環境。

答案： C

說明： 為 QA 測試複製生產環境，然後在測試完成後從 QA 提升到生產環境是應用程式開發環境中的常見設計實踐。將正在進行 QA 的軟體添加到生產環境或允許用戶使用 QA 系統而不是生產環境可能會導致負面的用戶體驗，創建與生產不匹配的 QA 環境可能會使測試失效。

  4. SANS/CWE
