1. Juanita 已將她的虛擬化集群配置為高可用性模式。虛擬化管理平面檢測到她的一個伺服器已失敗且不再發送心跳信息。如果節點無法重新啟動，集群將做什麼？

  1. 嘗試重新啟動失敗的系統。
  2. 關閉失敗的系統並發送警報。
  3. 在其他集群節點上重新啟動該系統上託管的 VM。
  4. 將主機從失敗的系統遷移到其備份鏡像。

答案： C

說明： HA 或高可用性模式使用心跳來檢測系統何時失敗。如果心跳停止，則該節點上運行的虛擬機器將在其他主機上重新啟動。

2. Will 希望在他的雲託管環境中使用容器化應用程式。在他構建它們時，他應該使用以下哪個最佳實踐？

  1. 每個容器打包一個應用程式。
  2. 盡可能使用默認安裝。
  3. 保留所有正常的工具和實用程序。
  4. 避免標記以降低複雜性。

答案： A

說明： Will 知道容器應每個容器打包一個應用程式。安裝應配置為滿足應用程式的需求，如果不需要，則應移除默認工具。標記對於輕鬆控制容器至關重要。

3. Jason 希望採用雲服務安全標準。以下哪個標準是專門設計用於涵蓋雲服務提供商的？

  1. ISO/IEC 20000:1
  2. ISO/IEC 27017
  3. PCI DSS
  4. GDPR

答案： B

說明： ISO/IEC 27017 是一個專門設計用於雲服務提供商的安全標準。ISO/IEC:20000:1 和 PCI DSS 是更廣泛的標準，歐盟通用數據保護條例也不是特定於雲的。

4. Helen 的組織運營一個由雲服務提供商託管的電子商務網站。她可能需要遵守以下哪個合規標準？

  1. PCI DSS
  2. FedRAMP
  3. COBIT
  4. ITIL

答案： A

說明： 由於 Helen 運營電子商務網站，她很可能接受信用卡，因此需要確保符合 PCI 標準——這是一項由合同而非法規要求的標準。FedRAMP 是一個美國聯邦計劃，側重於與雲服務相關的安全評估、監控和其他元素。COBIT 是一個定義 IT 管理實踐的框架，與 ITIL 非常相似。

5. Ilya 希望確保他的雲環境中的系統得到適當修補。如果他的組織非常強調使用預構建工具，以下哪個選項將給予他對補丁安裝時間和安裝內容的最大靈活性和控制權？

  1. 使用內建的操作系統工具自動安裝補丁。
  2. 使用由組織開發的補丁腳本。
  3. 為所有應用程式和操作系統設置自動更新。

說明： 像 Amazon 的 Patch Manager 這樣的工具使用補丁基線來確定哪些補丁將被安裝或不安裝以及何時安裝，還可以掃描以確定補丁狀態。這既符合 Ilya 對靈活性的需求，也符合組織對購買與構建類型工具的偏好。

6. Yasmine 正在驗證她的軟體的負載性能，包括測試比她組織預期使用的用戶數量更高的用戶。她正在進行哪種類型的測試？

7. Adam 的組織使用 Google 雲服務，他希望確保其組織的日誌是安全的。他應向其組織推薦哪個最佳實踐以確保日誌在靜態時是安全的？

說明： 使用客戶管理的金鑰加密日誌是確保日誌無法被第三方訪問的最佳實踐。Adam 應確保他的組織安全地控制和管理他們的金鑰。允許供應商控制金鑰意味著第三方，包括執法部門或侵入提供商安全的攻擊者，可能會訪問他組織的日誌，而不加密日誌意味著它們在靜態時不受保護。

8. Charleen 希望為她的組織實施多因素身份驗證。以下哪個 MFA 選項被認為最不安全？

  2. 基於硬體令牌的代碼生成
  3. 基於 SMS 的代碼傳遞
  4. USB 硬體令牌

9. 電子郵件和網頁都是哪種類型數據的範例？

10. Felix 希望確保其組織的成員僅在批准的位置訪問管理控制台。以下哪個網路安全能力最能幫助他完成此任務？

  3. 流量檢查
  4. 網路安全組

答案： B

說明： 地理圍欄允許組織指定特定位置以允許或拒絕操作。這將允許 Felix 將組織的設施設置為可接受的訪問位置，同時拒絕來自其他位置的訪問。零信任側重於持續授權和身份驗證，流量檢查著眼於流量以識別不需要或意外的流量，網路安全組的工作方式很像防火牆，根據規則允許或拒絕流量。

11. Valerie，一位 Amazon AWS 用戶，擔心可能影響她當前美東地區的潛在中斷。她應採取哪種風險緩解策略來確保其組織能夠應對區域範圍內的中斷？

  1. 在另一個區域配置 DR。
  2. 在另一個可用區配置 DR。
  3. 在她當前的可用區部署冗餘系統。
  4. 在她當前的區域部署冗餘系統。

答案： A

說明： Valerie 知道區域範圍內的中斷可能需要在庫存恢復之前在另一個區域進行操作。雖然這可能是一項重大的工作，但在真正的災難場景中可能是必要的。雖然她可以考慮另一個雲提供商，但從一個提供商轉移到另一個提供商所需的新工作量很大。僅選擇另一個可用區或在同一區域或區域部署更多資源無法滿足她的需求，因為如果該區域關閉，它們都會受到影響。

12. Casey 正在從本地數據中心過渡到雲數據中心。在她的新環境中，她仍然能夠訪問哪個硬體監控？

13. Kirk 正在為其組織採用平台即服務工具。在 PaaS 環境中，誰負責應用程式和數據安全？

  1. 客戶

  4. 客戶和提供商雙方

14. Alaina 希望保護她的本地數據中心免受電力問題的影響。以下哪個選項最適合處理電壓驟降？

  1. 發電機
  2. 從兩個不同的提供商處購買電力
  3. UPS
  4. PDU

答案： C

說明： 不間斷電源 (UPS) 系統使用電池或其他機制在電壓驟降時提供電力。發電機需要時間才能啟動，無法輕鬆處理短暫的中斷。從兩個不同的提供商處購買電力可能會有所幫助，但如果仍然發生電壓驟降，UPS 是更好的解決方案。配電單元 (PDU) 用於將電力輸送到伺服器和系統並對其進行控制和監控，但它們對電壓驟降或電力損失沒有影響。

15. 哪個術語用於描述允許客戶創建虛擬機器、使用虛擬網路定義自己的網路以及使用儲存和其他服務來創建和管理自己基礎設施的雲服務提供商？

  1. IaaS
  2. PaaS
  3. SaaS
  4. CaaS

答案： A

說明： 基礎設施即服務解決方案允許客戶使用服務提供商提供的資源構建和管理自己的基礎設施。PaaS 為客戶提供了一個可以在其上構建應用程式的平台，SaaS 提供軟體即服務。CaaS 有時代表容器即服務，但不是一個廣泛採用的雲術語。

16. 當 Susan 登錄其組織的服務門戶時，她看到客戶數據已刪除姓名和地址。她的組織正在使用哪種數據混淆技術？

  1. 隨機化
  2. 數據遮蔽
  3. 哈希
  4. 匿名化

說明： Susan 的組織正在使用匿名化從客戶數據中移除可識別資訊。隨機化會隨機化數據，移除其與其他元素的聯繫，同時保留格式和內容以供測試。數據遮蔽會從數據視圖中移除數據元素，通常使用星號或其他字符替換資訊。哈希使用哈希創建仍然可以引用的數據元素，而無需訪問原始數據。

17. Hui 希望進行一次涵蓋其組織安全實踐的時間點 SOC 審計。她應選擇哪種類型的 SOC 審計？

23. 什麼定義了 Type 2 hypervisor？

  2. 它運行在現有操作系統之上。
  3. 它安裝在裸機系統上。
  4. 它無法在虛擬化系統內部運行。

24. Amazon 的 S3 依賴於儲存桶中的對象集合，並可通過 REST API 訪問。Amazon S3 是哪種常見的雲儲存類型？

  1. 對象

  3. 原生
  4. 網路文件

25. 以下哪個 *不是* 隱私影響評估 (PIA) 的典型目標？

  1. 識別隱私工作的成本

  3. 識別隱私洩露的風險
  4. 識別隱私控制

說明： 隱私影響評估側重於個人身份資訊的內容、原因和方式，包括法律和政策要求、風險和控制。它不專門尋求識別隱私控制或工作的成本。

26. Mike 希望確保他的數據標籤隨著數據在其組織中使用而隨之移動。確保這一點的最有效和可行的選項是什麼？

  1. 在文件元數據中包含數據標籤。
  2. 在文件名中包含數據標籤。
  3. 將數據標籤作為與第一個文件一起發送的第二個文件包含在內。
  4. 將數據標籤作為密碼包裝的一部分包含在內。

說明： 以允許它們輕鬆跟隨文件的方式包含標籤是 Mike 的最有效選項。雖然標籤可能會被修改或移除，但正常使用不會導致它們丟失。文件名經常更改，第二個文件很容易被遺忘或移除，密碼包裝會引入開銷，並且也可能被解密和遺忘。

27. 在發現洩露後，哪個群體最有可能有法律規定的通知時間框架？

  1. 客戶
  2. 合作夥伴
  3. 監管機構
  4. 執法部門

答案： C

說明： 法規通常具有內建的法律規定的通知時間框架。它可能設定一個時間段，或者僅標記為要求合理的時間框架，但法規最有可能要求這樣做。客戶和合作夥伴可能有合同通知要求，而不是法律要求，執法部門通知通常由組織自行決定。

28. Google 的雲架構框架建議了許多常見的安全設計實踐。以下哪個 *不是* 常見的雲 IaaS 設計安全實踐？

  3. 強調手動監控。

答案： C

說明： 現代設計原則要求盡可能自動化監控。雲部署的範圍和規模以及日誌和其他監控資訊的量意味著手動工作不太可能成功。相反，應盡可能保留手動工作用於升級和監督。滿足合規要求、使用分層安全以及自動化敏感任務的部署都在 Google 雲架構框架的安全原則中，網址為 https://cloud.google.com/architecture/framework/security/security-principles。

30. Michelle 希望為其雲環境儲存和管理密碼金鑰。如果她正在選擇新的提供商，她應要求其雲 IaaS 提供商擁有哪個解決方案？

  1. TPM
  2. 雲 HSM
  3. PKI

125. Maria 登錄其公司的服務門戶並查看客戶的資訊。在信用卡字段中，她看到信用卡列出的格式如下：
XXXX-XXXX-XXXX-1980
正在使用哪種類型的數據安全技術？

  1. 哈希
  2. 遮蔽
  3. 隨機化
  4. 去識別化

答案： B

說明： Maria 的組織正在使用遮蔽技術來隱藏大部分信用卡號碼。保留最後四位數字是一種旨在允許驗證卡號的技術。哈希將導致一個固定長度的字母和數字字符串，作為單向函數的一部分。隨機化可能用於混合真實數據以用於示例數據集，去識別化則移除個人身份數據，例如地址、社會安全號碼或姓名。
