第二章 問題：架構與設計（含答案與說明）

1. 電子郵件是哪種類型數據的範例？

  1. 結構化數據
  2. 半結構化數據
  3. RFC 定義的數據
  4. 非結構化數據

答案： D

說明： 電子郵件和其他自由格式文本是非結構化數據的範例。像資料庫中的數據一樣的結構化數據是經過仔細定義的，而像 XML 和 JSON 一樣的半結構化數據則應用了結構但沒有嚴格控制。雖然電子郵件本身是由 RFC 定義的，但術語 RFC 定義的數據在此上下文中不使用。

2. Nick 希望確保數據一旦分類後能得到妥善處理。他知道數據標籤對這個過程很重要，並且將幫助他的數據丟失預防工具防止數據洩露和暴露。在他的數據生命週期中，應何時標記數據？

  1. 創建
  2. 儲存
  3. 使用
  4. 銷毀

答案： A

說明： 數據標籤通常發生在數據生命週期的創建階段。數據標籤也可能在數據使用過程中更改或添加，因為數據會被修改。

3. Jacinda 計劃在她的雲環境中部署數據丟失預防 (DLP) 系統。以下哪個挑戰最有可能影響她的 DLP 系統判斷敏感數據是否正在傳輸到其組織外部的能力？

  1. 缺乏數據標籤
  2. 使用傳輸中數據加密
  3. 數據標籤不當
  4. 使用靜態數據加密

答案： B

說明： Jacinda 很可能因為在雲環境中廣泛且一致地使用傳輸中數據或動態數據加密而面臨 DLP 系統的挑戰。她需要特別注意設計和架構她的環境，以允許 DLP 系統訪問其所需的流量。如果數據標籤缺乏或不當，這可能是一個挑戰，但 DLP 系統可以使用模式匹配和其他技術來識別數據。靜態數據通常對 DLP 系統來說不是問題，因為防止丟失需要了解數據何時流向某處，而不是它停留在某個位置。

4. Susan 希望確保她的雲環境中的超級用戶訪問能夠被妥善審計。以下哪項 *不是* 審計特權用戶訪問的常見要求項目？

  1. 遠程 IP 地址
  2. 使用的帳戶
  3. 使用的密碼
  4. 本地 IP 地址

答案： C

說明： 密碼是有意不被捕獲或記錄的，因為創建包含密碼的審計日誌將是一個重大的安全問題。源和目標 IP 地址以及用於特權訪問的帳戶都是常見的日誌數據，有助於審計事件。

5. Ben 的組織對其本地系統和第三方提供的雲託管系統使用相同的數據刪除程序。Ben 認為當前使用的過程存在問題，該過程涉及在重複使用之前對使用中的磁碟和卷執行單次零擦除。Ben 應該為雲環境強調此方法的哪個問題？

  1. 加密銷毀是第三方託管雲平台的安全選項。
  2. 僅零擦除是不夠的，還應使用隨機模式。
  3. 零擦除需要多次傳遞以確保沒有殘留數據。
  4. 應對硬碟進行消磁而不是擦除或加密銷毀，以確保數據在物理層面上完全銷毀。

答案： A

說明： 加密擦除或加密銷毀是確保第三方託管的硬碟和卷被安全清除的唯一方法。零擦除可能會留下殘留數據，尤其是在硬碟在託管環境中動態分配空間的情況下。消磁或其他物理銷毀通常對於第三方託管系統來說是不可能的，除非有特殊合同和專用硬體。

6. Jason 已被告知，他的組織需要對與未決訴訟相關的資訊實施法律保留。他應採取什麼行動來實施保留？

  1. 從備份中恢復文件，使其與保留請求的日期匹配。
  2. 搜索與訴訟相關的所有文件，並立即將其提供給對方律師。
  3. 刪除法律保留中指定的所有文件，以限制訴訟範圍。
  4. 識別範圍內的文件並將其保存起來，直到需要出示為止。

答案： D

說明： 法律保留要求組織識別並保存符合保留範圍的數據。Jason 應識別這些文件並將其保存起來，直到需要時為止。恢復文件可能會刪除重要數據，刪除文件完全違背了保留的概念，而且保留不需要立即出示——它們就像聽起來那樣，是保留數據的要求。

7. Murali 正在審查其組織客戶關係管理工具中的客戶文件，並看到客戶的社會安全號碼列為 XXX-XX-8945。使用了哪種數據混淆技術？

  1. 匿名化
  2. 遮蔽 (Masking)
  3. 隨機化
  4. 哈希

答案： B

說明： 數據遮蔽涉及用備用字符（如 X 或 *）替換數據。這通常是通過軟體或資料庫本身的控制來完成的，因為底層數據在資料庫中保持不變。匿名化或去識別化會刪除可能允許識別個人的數據。哈希用於允許通過哈希引用數據而不顯示實際數據，但它會導致可能需要用於測試的數據屬性丟失。隨機化或混淆數據會移動數據，使其去關聯，但將真實數據留在原地進行測試。

  1. 非結構化數據
  2. 重構數據
  3. 半結構化數據
  4. 結構化數據

9. Lucca 希望在基礎設施即服務雲服務提供商的環境中為他的 Linux 實例實施日誌記錄。他希望捕獲諸如系統創建和銷毀之類的事件，作為性能擴展要求的一部分。他應該使用哪個日誌記錄工具或服務來對這些事件獲得最深入的了解？

  1. 來自 Linux 系統的 Syslog
  2. 雲服務提供商的內建日誌記錄功能
  3. 來自 Linux 系統的 Syslog-NG
  4. 來自 Linux 系統的本地事件日誌和應用程式日誌

答案： B

說明： 提供商自己的日誌記錄功能是最佳選擇。關於系統創建和銷毀的資訊不會存在於本地系統上，因此 syslog、syslog-ng 和本地日誌都無法工作。此外，Linux 通常沒有應用程式日誌——事件和應用程式日誌都是 Windows 系統的常見日誌。

10. Joanna 的公司使用負載均衡器在多個 Web 伺服器之間分配流量。當流量通過雲環境中的負載均衡器傳遞到本地 Web 伺服器時，通常會丟失哪個數據點？

  4. 目標 IP 地址

11. Isaac 正在使用哈希函數進行完整性檢查，並允許引用地址數據而無需暴露實際數據。當數據被哈希時，數據的以下哪個屬性 *不會* 丟失？

  2. 數據的長度
  3. 數據的格式
  4. 根據街道號碼對數據進行排序的能力

答案： A

說明： 哈希將可變長度數據轉換為固定長度輸出，這意味著長度、格式以及使用字符串或數字對數據執行操作的能力都將丟失。其唯一標識的能力不會丟失——Isaac 只需要知道給定地址的哈希值即可繼續引用該數據元素。

12. Amanda 的安全數據儲存操作程序要求她確保使用數據分散技術。Amanda 需要做什麼才能符合此要求？

  1. 刪除所有不在安全儲存中的數據。

13. Lisa 在她的雲託管環境中運行 Windows 實例。每個 Windows 實例都創建有一個 C: 驅動器，用於存放操作系統和應用程式文件。哪種類型的儲存最能描述這些 Windows 實例的 C: 驅動器？

  1. 長期儲存
  2. 臨時儲存
  3. 原始儲存
  4. 基於卷的儲存

答案： B

說明： 與實例關聯並在實例關閉時將被銷毀的儲存是臨時儲存。原始儲存是指您可以直接訪問的儲存，例如具有底層設備訪問權限的硬碟或 SSD。長期儲存是指旨在持續存在的儲存，通常用於日誌或數據儲存。基於卷的儲存是指在雲中分配為虛擬驅動器或設備的儲存。

16. Gary 正在代表他的公司收集數據以支持法律案件。為什麼他可能會在收集文件時對其進行數字簽名，並將其與數據一起以經過記錄、驗證的方式保存？

17. Valerie 正在為她的雲環境執行風險評估，並希望識別她的組織用於可擴展、基於虛擬機的環境中系統驅動器的臨時卷儲存的風險。以下哪項 *不是* 臨時儲存的威脅？

  3. 因其臨時性質導致的性能不佳
  4. 鑑識產物的丟失

說明： 臨時儲存將具有其整體儲存類型的性能，因此性能不佳不是預期的問題。無意暴露、惡意訪問和鑑識產物的丟失都是臨時儲存的問題。

18. 在存儲被重新分配給其他客戶（如果未在解除分配時進行加密銷毀而僅進行零擦除）的環境中，哪種類型的存儲最有可能存在殘留數據問題？

  1. 臨時儲存
  2. 原始儲存
  3. 長期儲存
  4. 磁光儲存

答案： B

說明： 原始儲存提供對磁碟的直接訪問，如果不進行加密銷毀，使用後很可能在磁碟上留下殘留數據。由於長期儲存和臨時儲存通常是抽象的，因此在未分配或重新分配的扇區中留下殘留數據的可能性較小，這些扇區不會通過典型的擦除操作清除。
